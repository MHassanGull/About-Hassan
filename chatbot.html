01

Problem & Solution

The Problem

Static portfolios fail to engage visitors. People have specific questions about a developer's experience that aren't obvious from bullet points and project tiles. Visitors leave without answers — and without context, they leave without contacting.

The Solution

An n8n-orchestrated AI assistant using Retrieval-Augmented Generation. It searches a vector store of my portfolio knowledge, feeds that context to Claude, and replies in my voice — 24/7, with conversation memory, session tracking, and lead capture.

What actually happens, step by step

  1. Visitor sends a message → the widget on the portfolio site POSTs {message, sessionId} to the n8n webhook.
  2. Extract Input normalises the body, generates a session ID if missing, and forwards {chatInput, sessionId}.
  3. AI Agent assembles the prompt: hardcoded facts about Hassan, last 15 messages from Window Buffer Memory, plus access to the portfolio_knowledge tool.
  4. If the question needs depth, the Agent fires the tool — which embeds the query with HuggingFace all-mpnet-base-v2, queries Pinecone (hassan-website namespace), and feeds the top matches back to the Agent.
  5. Claude Haiku 4.5 synthesises the answer in Hassan's voice. The output fans out twice in parallel:
  6. Respond to Webhook ships {reply, success: true} back to the widget with proper CORS — visitor sees the reply.
  7. Save to Firebase simultaneously logs the conversation to chatbot/conversations RTDB for analytics — failures don't block the response.
03

Technologies

n8n

Self-hosted workflow orchestrator. Defines triggers, runs the agent, routes responses.

Claude Haiku 4.5

Anthropic's fast, capable model. Handles reasoning, tool-calling, and personality.

Pinecone

Vector database storing 19 hand-curated chunks of portfolio knowledge in 768 dims.

HuggingFace

sentence-transformers/all-mpnet-base-v2 for query embeddings — free, accurate.

Firebase RTDB

Realtime database persists every conversation for analytics and lead tracking.

Vanilla JS Widget

Lightweight floating bubble — fetch + sessionId persistence, no framework overhead.

Key Capabilities

  • Retrieval-Augmented GenerationVector search runs only when needed — fast answers stay fast.
  • Conversation memoryLast 15 turns stay in context per session — no repeating yourself.
  • Session persistenceBrowser-side sessionId survives reloads via localStorage.
  • Parallel side-effectsFirebase logging never delays the user-facing reply.
  • Lead intent detectionSystem prompt routes hiring questions to name/email capture.
  • Production CORSOrigin-restricted headers — only the portfolio domain can call it.
  • Graceful degradationNetwork errors render an inline retry message, never a broken bubble.
  • Zero frameworkVanilla JS widget — <3KB minified — works on any static site.
