Section 01
How It Works
A customer phone call travels through four systems in under two seconds — from the caller's voice to the kitchen screen.
01
Customer Calls the Restaurant
The customer dials the restaurant phone number. The call routes via PSTN to Retell AI's platform, which answers instantly — no ringing, no waiting, no hold music.
02
Hassan Listens and Understands
Retell's Speech-to-Text converts the caller's voice to text in real time. The LLM processes the text, understands what the customer wants — order, track, update, or cancel — and manages the full back-and-forth conversation naturally.
STT → LLM → TTS
03
Retell Fires an API Tool Call
When Hassan needs to place or modify an order, Retell fires an HTTP POST request to our FastAPI backend with the extracted customer data — name, phone number, items ordered, and any special requests.
POST /createorder/ → FastAPI
04
FastAPI Writes to Firebase
The async route handler validates the data, checks for duplicate orders by phone number, and writes the new order document to Firebase Firestore with a server timestamp and pending status.
FirestoreDB.create_order()
05
Kitchen Dashboard Updates Live
Immediately after writing to Firestore, FastAPI broadcasts a WebSocket message via the ConnectionManager to all connected kitchen browsers. A new row appears on the kitchen screen in real time — no refresh needed.
manager.broadcast() → wss://
06
Hassan Confirms to the Customer
FastAPI returns {"success": true} to Retell. The LLM reads the response and Hassan speaks the confirmation — "Your order is in, ready in about 15 minutes!" — and wraps up the call naturally.
{"success": true} → TTS → Customer
Section 02
Technology Stack
Four carefully chosen technologies, each handling exactly what it does best.
📞
Retell AI
Voice Layer
Retell is a voice AI platform that handles everything about the phone call. It manages speech-to-text conversion, runs an LLM to power the conversation, converts text back to natural speech, and triggers HTTP tool calls to our backend when action is needed.
Why Retell? It abstracts all telephony complexity. We write zero speech processing code — only the business logic.
⚡
FastAPI
API Layer
FastAPI is a modern Python web framework built for speed. It is natively asynchronous, which means it handles multiple requests simultaneously without blocking. Retell calls our backend on every turn of a live phone call — if our API is slow, the customer hears an awkward silence.
Why FastAPI? 3-5x faster than traditional frameworks. Native WebSocket. Pydantic validation built-in. One main.py replaces dozens of files.
🔥
Firebase Firestore
Data Layer
Firestore is a NoSQL cloud database by Google. Every order is stored as a document in the "orders" collection with fields for name, phone, items, special requests, status, and timestamps. Phone number is the lookup key for all order queries.
Why Firestore? Zero migrations, zero server management, scales automatically. The firebase-admin SDK connects in 3 lines of Python.
🖥️
Kitchen Dashboard
Real-Time UI Layer
The kitchen dashboard is an HTML page served by FastAPI using Jinja2. On page load, all current orders are rendered server-side. After that, a native FastAPI WebSocket connection keeps it updated live — new orders appear instantly, status changes reflect immediately across all screens.
Why native WebSocket? No Redis, no external broker. A simple in-memory ConnectionManager handles all real-time broadcasts.

Section 04
Key Features
What makes this system production-ready.
🎙️
Natural Voice Conversation
Hassan speaks like a real person. Warm, casual, asks one question at a time, confirms before placing, and handles edge cases gracefully — all powered by Retell's LLM.
⚡
Sub-Second API Response
FastAPI's async architecture ensures Retell gets a response fast enough that the customer never hears a pause. Slow APIs break the voice experience.
🔄
Real-Time Kitchen Updates
Native WebSocket broadcasting with ConnectionManager. New orders appear on every kitchen screen the moment they are placed, with zero page refresh.
🛡️
Payload Envelope Handling
Retell wraps request data differently across call types. A custom extract_payload() helper automatically unwraps arguments, data, or payload envelopes every time.
☁️
Zero-Server Deployment
Fully containerised with Docker and deployed on Hugging Face Spaces. Git-push deploys. HTTPS automatic. Secrets managed via HF encrypted environment variables.
📦
Minimal Project Structure
The entire backend is one main.py file plus firebase_utils.py and schemas.py. No app folders, no migrations, no boilerplate. Easy to read, easy to maintain.
